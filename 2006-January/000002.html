Anyway - I have another suggestion. This may already exist, if it does I
apologise - but I'm not aware of it.

How about a "documentation TODO list" ? This could be a list of features
that are undocumented or poorly documented.

As you add new features but don't have time to document them you can add
them to this list. Volunteers (like myself) can then look at the list
and pick an item to work on. This breaks the documentation task down
into smaller amounts. I quite enjoy writing documentation - so if I knew
*specifically* what features needed writing, then I would have a go.

Of course the list would need maintaining - but once it is created that
is a *relatively* trivial task.
